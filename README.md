## ⚡ Executive Overview

> **Leviathan non è un semplice addon.**  
> È un layer operativo progettato per aggregare, filtrare, ordinare e presentare risultati con una logica più vicina a un protocollo che a uno scraper tradizionale.

Il progetto nasce per offrire una base tecnica capace di:

- unire **sorgenti torrent**, **provider web** e **moduli ibridi**
- ridurre il rumore nei risultati
- migliorare il matching semantico
- mantenere una latenza percepita aggressiva
- restituire output leggibili e pronti all’uso in ambiente Stremio

---

## 🔥 Release Highlights

- 🚀 **Core Refactoring** — motore riorganizzato per maggiore stabilità, leggibilità e tenuta sotto carico.
- 🌐 **WebStreams Auto-Failover** — attivazione automatica della chain web quando il layer torrent non produce output utili.
- 🎨 **Polymorphic Formatter Engine** — rendering più pulito, gerarchico e leggibile dentro Stremio.
- 🗣️ **Tri-Scope Language Control** — modalità dedicate per **ITA**, **ENG** e **Hybrid**.
- 🌪️ **VIX Hybrid Module** — integrazione con sorgenti web ad avvio rapido.
- 👻 **Ghost Proxy Compatibility** — supporto MediaFlow / ambienti proxy-based.
- 📡 **Direct Swarm Protocol** — riproduzione P2P diretta per scenari senza debrid.

### Un solo blocco operativo, più layer sincronizzati

**Leviathan** concentra la propria logica in una pipeline unica che combina **validazione semantica Italy-First**, **routing a latenza adattiva**, **resilienza contro challenge e nodi degradati**, **fusione intelligente dei metadati** e **delivery ibrido torrent + web**.

In pratica il protocollo analizza pattern come `MULTI`, `SUB-ITA`, `AC3` e `DTS`, distingue release realmente italiane da risultati ambigui, riduce i falsi positivi e ordina le sorgenti con priorità più sensate per l’utente finale. Sul piano operativo decide dinamicamente quali provider meritano la corsia veloce e quali richiedono una scansione più profonda, bilanciando velocità, qualità del matching e continuità del flusso.

A livello infrastrutturale integra **WAF handling**, **identity rotation**, **failover automatici**, **magnet enrichment** e un layer ibrido con **StreamingCommunity**, **GuardaHD** e **GuardaSerie**, così da poter passare in modo trasparente dal percorso torrent a quello web quando serve un avvio immediato o quando il P2P non restituisce un risultato abbastanza solido.

Il blocco viene completato da **Debrid Ghost Shell** per scenari proxy-based, **WebStreams Auto-Failover** per evitare vicoli ciechi, **Polymorphic Formatter Engine** per una resa visiva superiore, **Linguistic Scope Control** per gestire ITA / ENG / Hybrid, **Trailer Bridge** per le anteprime contestuali e **Direct Swarm Access** per la riproduzione P2P pura.

**Risultato finale:** un sistema coerente progettato per restituire output **più puliti**, **più rapidi**, **più credibili** e molto più **leggibili**.

# ⚠️ Self-Hosting Reality Check

> [!IMPORTANT]
> **Leviathan può essere self-hostato, ma bisogna capire bene cosa significa.**

L’istanza pubblica del progetto non si limita al solo codice del repository. Parte dell’esperienza reale dipende anche da una componente dati e da una struttura operativa che **non è inclusa** nel repository open-source.

## Cosa non è incluso

- **il database della istanza pubblica**
- **eventuali cache pre-costruite lato server**
- **l’infrastruttura dati privata usata per ottimizzare i risultati**
- **i vantaggi di warm cache, storico e tuning della versione live**

In altre parole: il repository contiene il motore, ma **non l’intero ecosistema operativo** della deployment pubblica.

## Cosa significa in pratica

In self-hosting, Leviathan deve lavorare in modo più grezzo:

- più scraping live
- meno dati già pronti
- più dipendenza dalla qualità della tua rete e della tua VPS
- esperienza diversa rispetto alla live instance

## Conviene self-hostarlo?

Per studio, sviluppo, debugging o fork: **sì**.  
Per avere l’esperienza migliore da semplice utente finale: **no, nella maggior parte dei casi non conviene**.

| Scenario | Consiglio |
| :-- | :-- |
| Vuoi usare Leviathan al meglio | **Usa l’istanza pubblica** |
| Vuoi studiare o modificare il codice | **Self-hosting ok** |
| Vuoi le stesse performance della live instance | **No** |
| Vuoi il database della versione pubblica | **Non è incluso** |
